Pagination

Use this to manage multiple pages of content for example lists or tables of data.

Import Module

To use the components here you first need to include this module in your application @NgModule

Attributes

The following table shows the various attribute bindings that can be used with this component.

Binding Default Required Description
[records] - Yes The total number of records that will be paginated through.
[skip] - Yes How many records are being skipped over - where should the pagination start from, default is the first page (Record 0).
[limit] 20 No How many records are being shown on each page that the pagination is managing.
[choices] 5 No How many page choices should there be in the pagination, recommended that this is an odd number.
[controls] true No Should the page jumping controls (previous, next, start, end) be shown to the user.
[icons] true No When using the controls, should we show text or icons to the user.
[size] normal No This allows limited control of the size for each pagination option, the sizes are: small, normal and large.
(change) - No A function to be called when an pagination event is triggered. Event contains the following fields;
  • skip: number
  • limit: number
  • currentPage: number
  • totalRecords: number
  • selectedPage: number
  • selectedSkip: number
